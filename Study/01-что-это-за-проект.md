# 1. Что это за проект

## Идея

**Checkin** — бэкенд для мобильного/веб-приложения «чек-инов по QR-кодам».
Логика похожа на коллекционную карту с печатями:

- **Места** (кафе, музеи, магазины) генерируют свой уникальный **QR-код**.
- **Пользователь** приходит в место, сканирует QR и получает **штамп** (stamp) —
  отметку о посещении.
- У каждого пользователя копится коллекция штампов (`stamps_count`).

Это типичный сценарий для программ лояльности, городских квестов, ивентов.

## Роли пользователей

В системе одна таблица пользователей, но две роли (поле `role`):

| Роль   | Значение | Что может делать                                            |
|--------|----------|-------------------------------------------------------------|
| `subj` | субъект  | Обычный пользователь: сканирует QR, собирает штампы.        |
| `obj`  | объект   | Владелец места: создаёт места, видит статистику, QR-коды.   |

Роль проверяется прямо в эндпоинтах (например, создавать место может только `obj`,
см. [places.py](../app/api/v1/endpoints/places.py)).

## Доменная модель (сущности БД)

Все модели описаны в одном файле — [app/models/user.py](../app/models/user.py).
(Название файла историческое; по-хорошему стоит разбить на `user.py`, `place.py`,
`stamp.py` — см. TODO в файле 05.)

```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│    User     │ 1     N │    Place     │ 1     N │    Stamp    │
│  (subj/obj) │────────<│  (с QR-кодом)│────────<│   (чекин)   │
└─────────────┘ owner   └──────────────┘ place   └─────────────┘
       │                       │ N                       │
       │ referred_by           │ category_id             │ user_id
       │ (само-ссылка)         ▼ 1                        │
       │                ┌──────────────┐                  │
       └────────────────│   Category   │                  │
         1 пользователь └──────────────┘   1 пользователь─┘
         ставит N штампов                  делает N штампов
```

### User — пользователь
Ключевые поля: `name`, `email` (уникальный), `phone`, `hashed_password`,
`role`, `referred_by` (кто пригласил — ссылка на другого `User`),
`stamps_count` (счётчик штампов), `is_active`.

### Place — место с QR-кодом
Принадлежит пользователю-`obj` (`owner_id`). Содержит `qr_code` (генерируется
как UUID), `stamp_url` (картинка печати), `category_id`, `is_active`.

### Stamp — штамп (факт чек-ина)
Связывает `user_id` ↔ `place_id`. Есть уникальное ограничение
`uq_stamp_user_place`: один пользователь может отметиться в одном месте
**только один раз**.

### Category — категория места
Справочник (кафе, музей и т.п.). Простая таблица `id` + `name` + `description`.

## Реферальная система

При регистрации можно передать `?ref=<id пригласившего>`. Тогда у нового
пользователя заполнится `referred_by`. Свою реферальную ссылку пользователь
получает через `GET /api/users/referral_link/`.

> Пока это просто учёт «кто кого привёл» — никаких бонусов за рефералов в коде ещё нет.
