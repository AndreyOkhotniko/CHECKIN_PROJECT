# 3. Как запустить локально

> Команды даны для Windows (PowerShell). Аналоги для Linux/Mac — в комментариях.

## Предварительные требования
- Python 3.9+
- Docker Desktop (для PostgreSQL и pgAdmin) — либо свой PostgreSQL.

## Пошагово

### 1. Виртуальное окружение
```powershell
python -m venv venv
venv\Scripts\Activate.ps1     # Linux/Mac: source venv/bin/activate
```

### 2. Зависимости
```powershell
pip install -r requirements.txt
```

### 3. Файл окружения
Скопируй шаблон и заполни значения:
```powershell
copy .env.example .env         # Linux/Mac: cp .env.example .env
```
Открой `.env` и **обязательно поменяй `SECRET_KEY`** на случайный.
Сгенерировать можно так:
```powershell
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

> ⚠️ Если `.env` не загружается, проверь [config.py](../app/core/config.py):
> для pydantic-settings конфиг должен задаваться через `SettingsConfigDict`
> (а не `ConfigDict`), иначе параметр `env_file` молча игнорируется.

### 4. База данных (Docker)
```powershell
docker-compose up -d
```
Поднимется:
- PostgreSQL на `localhost:5432`,
- pgAdmin на `http://localhost:5050` (логин/пароль из `.env`).

### 5. Миграции
Сейчас в репозитории **нет ни одной миграции** (папка `alembic/versions/`
отсутствует). Их нужно сгенерировать один раз:
```powershell
alembic revision --autogenerate -m "Initial schema"
alembic upgrade head
```
Alembic использует синхронный драйвер `psycopg2` — он есть в `requirements.txt`.

### 6. Запуск приложения
```powershell
uvicorn app.main:app --reload
```
Приложение поднимется на `http://localhost:8000`.

## Проверка, что всё работает

| URL                              | Что это                                   |
|----------------------------------|-------------------------------------------|
| http://localhost:8000/           | Корневой эндпоинт (название, окружение)    |
| http://localhost:8000/health     | Проверка живости + подключения к БД        |
| http://localhost:8000/docs       | **Swagger UI** — интерактивная документация API |
| http://localhost:8000/redoc      | Альтернативная документация (ReDoc)        |

Начни с `/docs` — там видны все эндпоинты, можно прямо из браузера
зарегистрироваться, войти и подёргать API.

## Быстрая проверка без БД

Импорт приложения не требует подключения к БД (движок создаётся лениво):
```powershell
python -c "import app.main; print('routes:', len(app.main.app.routes))"
```
Если выводит число маршрутов без ошибок — импорты и конфиг в порядке.
