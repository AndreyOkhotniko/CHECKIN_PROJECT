# 5. Состояние проекта, внесённые правки и TODO

## Что уже реализовано ✅

- Аутентификация на JWT: регистрация, вход, refresh, защита эндпоинтов.
- Хеширование паролей (bcrypt).
- Профиль пользователя (просмотр/редактирование) + реферальная ссылка.
- CRUD мест с генерацией QR-кода и проверкой прав владельца.
- Постановка штампов, превью по QR, коллекция штампов, статистика по месту.
- Офлайн-синхронизация чек-инов пачкой.
- Справочник категорий.
- Инфраструктура для локального запуска (docker-compose: Postgres + pgAdmin).

## Чего ещё нет ❌ (главные пробелы)

| Пробел | Почему важно |
|--------|--------------|
| **Нет миграций Alembic** (`alembic/versions/` пустая) | Без них схему БД нельзя развернуть штатно. Нужно сгенерировать первую миграцию — см. файл 03, шаг 5. |
| **Нет тестов** | Любая правка может незаметно что-то сломать. Стоит завести `pytest` + `httpx.AsyncClient`. |
| **Все модели в одном файле** `models/user.py` | Неудобно расширять. По-хорошему разнести на `user.py`, `place.py`, `stamp.py`, `category.py`. |
| **Нет пагинации** в списках мест/штампов | На больших объёмах ответы станут тяжёлыми. |
| **Пустой файл** `app/core/database.py` | Артефакт, вводит в заблуждение. Можно удалить. |
| **Нет CORS-настроек** | Понадобятся, когда подключат фронтенд/мобайл. |

## Какие проблемы были исправлены (аудит) и почему

Ниже — правки, внесённые при ревью. Полезно понимать **почему** так, а не «как было».

### Безопасность и репозиторий
1. **`venv/` и `__pycache__/` убраны из git** (было закоммичено ~4000+ файлов).
   Виртуальное окружение и байткод платформозависимы и не должны быть в репозитории.
   Добавлен полноценный [.gitignore](../.gitignore).
2. **`.env` убран из git** и добавлен [.env.example](../.env.example).
   В `.env` лежат секреты (ключ JWT, пароли БД) — они не должны попадать в историю git.
   > ⚠️ Если репозиторий уже куда-то пушился со старым `.env` — секреты нужно **ротировать**
   > (сменить `SECRET_KEY` и пароли), потому что они остались в истории коммитов.
3. **Вход (`login`) переведён на тело запроса.** Было: `email`/`password` как
   query-параметры → пароль утекал в URL, логи сервера, историю браузера.
   Стало: схема `LoginRequest` в теле POST. См. [auth.py](../app/api/v1/endpoints/auth.py).

### Баги, которые ломали работу
4. **`config.py`: `ConfigDict` → `SettingsConfigDict`.** pydantic-settings читает
   `env_file` только из `SettingsConfigDict`. С обычным `ConfigDict` параметр молча
   игнорировался → `.env` не загружался → приложение работало на дефолтах (включая
   небезопасный `SECRET_KEY`). См. [config.py](../app/core/config.py).
5. **Возвращён `psycopg2-binary` в зависимости.** Его удалили, но Alembic использует
   синхронный `SYNC_DATABASE_URL` (`postgresql://...`), которому нужен psycopg2 —
   иначе миграции падают с `ModuleNotFoundError`. См. [requirements.txt](../requirements.txt).
6. **Валидация `ref` при регистрации.** Было `int(ref)` на сырой строке → нечисловой
   `ref` ронял запрос в 500. Стало `ref: Optional[int]` — FastAPI валидирует сам (422).

### Логика и целостность данных
7. **QR-коды убраны из публичной выдачи.** Эндпоинты `GET /api/places/` и
   `GET /api/places/{id}/` отдавали `qr_code`. Любой мог взять QR из публичного
   списка и поставить штамп, **не посещая место** — это ломает саму идею чек-ина.
   Введена схема `PlacePublicResponse` (без QR); полный `PlaceResponse` с QR
   отдаётся только владельцу. См. [schemas/place.py](../app/schemas/place.py).
8. **Добавлено уникальное ограничение `uq_stamp_user_place`** на пару
   (user_id, place_id). Раньше оно было только закомментировано — дубли штампов
   проходили при гонке запросов и при офлайн-синхронизации. См. [models/user.py](../app/models/user.py).
9. **`offline_sync` сделан устойчивым:** дедуп дублей внутри пакета + запись каждого
   штампа в своём savepoint (`begin_nested`), чтобы ошибка одного элемента не
   откатывала всю пачку. См. [stamps.py](../app/api/v1/endpoints/stamps.py).
10. **`verify_token` ловит больше ошибок.** Было `except JWTError` — кривой `sub`
    давал `TypeError`/`ValueError` и 500 вместо 401. Стало `except (JWTError, TypeError, ValueError)`.

### Мелочи
11. `health_check` снова реально проверяет БД (`SELECT 1`), а не возвращает «healthy» всегда.
12. Убран неиспользуемый `import uuid` в `users.py`.
13. `Place.is_active == True` → `Place.is_active.is_(True)` (корректный SQL-стиль).
14. Захардкоженный `https://yourapp.com` в реферальной ссылке заменён на
    настройку `BASE_URL` из конфига.

## Рекомендованные следующие шаги (для junior)

1. Сгенерировать и закоммитить первую миграцию Alembic.
2. Поднять окружение по файлу 03 и пройти сценарии из файла 04 через `/docs`.
3. Написать первые тесты на auth (регистрация → вход → `/me`).
4. Разнести модели по отдельным файлам.
5. **Сменить `SECRET_KEY`** в своём `.env` на случайное значение (не использовать дефолт).

> ⚠️ После добавления уникального ограничения на штампы (п.8) — не забудь, что
> для уже существующей БД его применит **новая миграция Alembic**. На чистой БД
> оно создастся сразу.
